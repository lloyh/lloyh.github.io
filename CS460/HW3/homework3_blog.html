Coding in C#

Given that my lower division programming course work background has been primarily in C++ and the fact that I have never programmed in Java, fully understanding the Java source code for this assignment took me some time and a good number of hours of researching. Once I fully understood the code, I proceeded to learning the basics of C# and started implementing the necessary classes class declarations and interface as outlined in the Java code.

First, I created a templated class called Node. It constructs a generic node that holds a data value and a pointer to the next node. If the node is null, this indicates the node is the head of the chain of nodes.

Second, I coded the LinkedStack class and its interface ILinkedStack.cs. This class creates a linked list, adds and removes nodes to and from the list, checks if the list is empty, returns the top of the stack (the last node added to the linked list) and resets the linked list.

The most challenging part of this C# code was Program.cs. The program first presents a title introduction and a brief instruction to enter a valid postfix expression to evaluate. It then creates a linked stack of doubles called “stack.” The while loop runs again and again until the user enters “q.” The loop first clears the stack, reads the user input as a string, trims any white spaces before and after the string entered (if any) and parses the string entered placing each character in a string array called “input.” Any number of spaces entered between characters is ignored via the Regex method:

If the first element of the input string array is not “q” the program proceeds to identify if the input is an operand, if it is it pushes it onto the stack until the first operator is identified. If neither an operand or operator is found, an error message is prompted, and the loop restarts and clears the stack. The program also evaluates if the postfix expression is valid by providing an error message if the stack becomes empty before the end of the loop. This would indicate there was not a proper number of operands vs operators or that the order in which there were entered is not correct in accordance with postfix notation.
