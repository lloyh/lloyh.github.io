For this assignment, I created an ASP.NET MVC 5 multi-page web application. It uses language constructs from the Razor view engine and utilizes C# language features such as namespaces, classes and methods. The assignment requirements can be accessed at this link: CS 460 Homework 4 Page.

Git Set Up and Starting a New MVC Project

I began this assignment by creating a new git branch called “hmw4-initial_setup” and a .gitignore file tailored to MVC projects in Visual Studio from the following github source: .gitignore for Visual Studio This file ignores all temporary build files as well as built binary ones.

I then proceeded to create a new MVC project in Visual Studio. As outlined by the assignment definitions, I made a “Home” default landing page (Index.cshtml) that contains Razor links to each of the pages outlined via @Html.ActionLink in an ul list.


Page 1: A Temperature Converter Using HTTP Method GET

After creating and checking out a new git branch for this feature, I added to my project a new page called Page1.cshtml. In it, I made a form that takes two inputs and submits the data to the Controller (HomeController.cs) via a GET Method. This method transmits the data via the web URL and thus it is cashed by the web browser.

The method for this page defined in the controller, takes the temperature and unit strings via the GET method and first validates the input. If the input is valid, it processes the data by converting the temperature into a double and checks for the unit value to be a letter F or f for Fahrenheit and C or c for Celsius. For either valid or invalid input, the appropriate message is presented on the same webpage (at the bottom of the page) via a ViewBag response string called “resultString.”

Note: the HTTP GET method passes the string form data via the URL on the address bar.


Page 2: A Temperature Converter Using HTTP Method POST

For Page 2, I used the same validation and processing code as the page also does temperature conversions, but the data is transmitted to the server via a POST method. Unlike the GET method, post does not pass data via the URL, instead it submits data to be processed to a specified resource contained in the HTTP message body.


Page 3: A Loan Calculator

For the third page, I created a loan calculator that takes three inputs: loan amount, APR and loan term in years. This data is also passed via a POST method and it is processed by the controller via model binding with parameters.
